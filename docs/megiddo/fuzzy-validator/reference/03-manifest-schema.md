# Fuzzy Validator — Manifest Schema

---

## 1. Page registry — `manifests/pages.json5`

Master list of pages to capture. The Playwright spec iterates this file.

```json5
{
  "defaults": {
    "viewport": { "width": 1280, "height": 720 },
    "repeat": 5,
    "waitAfterMs": 500,
    "auth": "none"
  },
  "pages": [
    {
      "id": "home-anonymous",
      "url": "./index.php?Route=",
      "auth": "none",
      "readySelector": "body",
      "notes": "Public home / event search widget"
    },
    {
      "id": "home-authenticated",
      "url": "./index.php?Route=",
      "auth": "login",
      "readySelector": "#main, body",
      "notes": "Post-login home; RSVP widget, kingdom chrome"
    },
    {
      "id": "player-profile",
      "url": "./index.php?Route=Player/profile",
      "auth": "login",
      "waitAfterMs": 1000,
      "notes": "T-01 / DS-09 player profile"
    }
  ]
}
```

### Field reference

| Field | Required | Description |
|-------|----------|-------------|
| `id` | yes | Stable slug; `[a-z0-9-]+`; keys baselines and manifests |
| `url` | yes | Path relative to `ORK3_E2E_BASE_URL` |
| `auth` | no | `none` (default) or `login` |
| `viewport` | no | Override `{ width, height }` |
| `repeat` | no | Calibration capture count (default from `defaults`) |
| `waitAfterMs` | no | Sleep after networkidle before screenshot |
| `readySelector` | no | Playwright locator wait before capture |
| `skip` | no | If `true`, excluded from `--all` unless explicitly named |
| `notes` | no | Human documentation only |

### Auth values

| Value | Behavior |
|-------|----------|
| `none` | Direct `page.goto(url)` |
| `login` | Run shared login flow; skip test if `ORK3_E2E_*` unset |

## 2. Database profiles — `manifests/profiles.json5`

Maps fuzzy-validator profiles to `bin/ork-db use` targets and **per-profile score thresholds**. See [11-dual-database-profiles.md](./11-dual-database-profiles.md).

| Profile | `orkDbUse` | DB | Strictness |
|---------|------------|-----|------------|
| `test` | `dev` | `ork_test` @ 19307 | **Strict** — `visualMinScore` 1.0 |
| `mirror` | `prod` | `ork` @ 19306 | **Lenient** — `visualMinScore` 0.98 |

Example: [examples/profiles.json5.example](./examples/profiles.json5.example)

Baselines and fuzz manifests live under **`manifests/{profile}/`** and **`baselines/{profile}/`**, not at the root of those trees.

## 3. Defaults — `manifests/defaults.json5`

Shared thresholds for Python tools.

```json5
{
  "colorThreshold": 20,
  "minAreaPx": 64,
  "padPx": 4,
  "calibrationRuns": 5,
  "gateColorThreshold": 20,
  "gateMaxOutsideDiffPx": 500,
  "morphologyKernelPx": 5,
  "assetsMinScore": 1.0,
  "domMinScore": 1.0,
  "visualMinScore": 1.0,
  "assetStripQuery": false,
  "domCompareScriptBodies": false
}
```

| Field | Description |
|-------|-------------|
| `colorThreshold` | 0–255 per-channel diff; pixel volatile if exceeded |
| `minAreaPx` | Minimum connected component area to become a fuzz box |
| `padPx` | Expand each box by this many pixels |
| `gateMaxOutsideDiffPx` | Secondary cap on absolute outside-diff pixels (visual layer) |
| `assetsMinScore` | Pass threshold for CSS/JS layer — default **1.0** (hard) |
| `domMinScore` | Pass threshold for DOM tree layer — default **1.0** |
| `visualMinScore` | Pass threshold for pixel layer — default **1.0**; tune to **0.98** if flaky |
| `morphologyKernelPx` | OpenCV dilate kernel size |
| `assetStripQuery` | If true, strip `?…` from asset URLs when hashing (Phase 2) |
| `domCompareScriptBodies` | If true, include inline script text in DOM tree (default false — assets gate owns scripts) |

Global defaults apply when a profile does not override. Profile thresholds in `profiles.json5` take precedence during `validate`.

## 4. Pixel fuzz manifest — `manifests/{profile}/{pageId}.fuzz.json`

Generated by `discover_fuzz.py`; committed after human review of overlay.

```json
{
  "schemaVersion": 1,
  "pageId": "home-authenticated",
  "imageWidth": 1280,
  "imageHeight": 3840,
  "calibratedAt": "2026-07-07T18:30:00Z",
  "calibratedFromCommit": "abc1234",
  "calibrationRuns": 5,
  "params": {
    "colorThreshold": 20,
    "minAreaPx": 64,
    "padPx": 4
  },
  "fuzzZones": [
    {
      "x": 1020,
      "y": 8,
      "width": 250,
      "height": 36,
      "source": "auto",
      "label": "optional human note"
    }
  ],
  "manualZones": []
}
```

### Field reference

| Field | Description |
|-------|-------------|
| `schemaVersion` | Increment on breaking schema changes |
| `imageWidth` / `imageHeight` | Dimensions at calibration time; gate fails on mismatch |
| `calibratedFromCommit` | Git SHA when fuzz was learned (optional but recommended) |
| `fuzzZones` | Auto-discovered rectangles in **screenshot pixel coordinates** (origin top-left) |
| `manualZones` | Hand-edited boxes merged at gate time (v1: edit JSON; v1.1: CLI add) |
| `source` | `auto` \| `manual` |
| `label` | Optional reviewer comment |

### Coordinate system

- Origin: top-left of full-page screenshot
- Units: CSS pixels at capture viewport width
- Boxes are axis-aligned; no rotation in v1

### Gate merge rule

Effective fuzz set = `fuzzZones` ∪ `manualZones`. Overlapping boxes are unioned before subtracting from diff mask.

## 5. Asset baseline — `baselines/{profile}/{pageId}.assets.json` (Phase 2)

Hard gate manifest. All calibration runs on a stable commit must produce **identical** content before committing.

```json
{
  "schemaVersion": 1,
  "pageId": "home-authenticated",
  "capturedAt": "2026-07-07T19:00:00Z",
  "capturedFromCommit": "abc1234",
  "assets": [
    {
      "id": "css-000",
      "kind": "css",
      "url": "http://localhost:19080/orkui/template/revised-frontend/script/revised.css",
      "inline": false,
      "sha256": "e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855",
      "byteLength": 482901,
      "baselinePath": "baselines/assets/home-authenticated/css-000-revised.css"
    },
    {
      "id": "inline-style-0",
      "kind": "css",
      "url": null,
      "inline": true,
      "sha256": "…",
      "byteLength": 128,
      "baselinePath": "baselines/assets/home-authenticated/inline-style-0.css"
    },
    {
      "id": "js-002",
      "kind": "js",
      "url": "http://localhost:19080/orkui/template/revised-frontend/script/revised.js",
      "inline": false,
      "sha256": "…",
      "byteLength": 912004,
      "baselinePath": "baselines/assets/home-authenticated/js-002-revised.js"
    }
  ]
}
```

### Gate rules

- Candidate manifest asset **id set** must equal baseline exactly.
- Each `sha256` must match **byte-for-byte**.
- No fuzz allowances on assets during refactor testing.

## 6. DOM fuzz manifest — `manifests/{profile}/{pageId}.dom-fuzz.json` (Phase 2)

Generated by `discover_dom_fuzz.py`; committed after review of debug report.

```json
{
  "schemaVersion": 1,
  "pageId": "home-authenticated",
  "calibratedAt": "2026-07-07T20:00:00Z",
  "calibratedFromCommit": "abc1234",
  "calibrationRuns": 5,
  "fuzzNodes": [
    {
      "path": "/html[0]/body[0]/header[0]/span[2]",
      "mode": "text",
      "source": "auto",
      "label": "session greeting"
    },
    {
      "path": "/html[0]/body[0]/form[0]/input[0]",
      "mode": "attributes",
      "attrs": ["value", "data-csrf"],
      "source": "auto"
    },
    {
      "path": "/html[0]/body[0]/div[3]",
      "mode": "subtree",
      "source": "auto",
      "label": "live weather widget"
    }
  ],
  "manualNodes": []
}
```

### `mode` values

| Mode | Gate ignores |
|------|--------------|
| `text` | Text content at path and descendant text nodes under that text leaf |
| `attributes` | Listed attribute names on the element at `path` (values may differ) |
| `subtree` | Entire node and all descendants (structure and content) |

### Path format

Index path from document root: `/html[0]/body[0]/div[2]/input[0]`. Assigned during canonicalization — see [05-phase2-asset-dom-gate.md](../archive/05-phase2-asset-dom-gate.md) §4.2.

Effective fuzz set = `fuzzNodes` ∪ `manualNodes`. Ancestor `subtree` fuzz subsumes child rules.

## 10. Versioning policy

| Change | Action |
|--------|--------|
| New page | Add to `pages.json5`; `record` for **both** profiles; commit `baselines/{profile}/` + manifests |
| Refactor with no frontend intent | `validate` must pass **test** (strict) and **mirror** (lenient) |
| Sandbox template change | Re-`record --profile test` only |
| Mirror dump refresh | Re-`record --profile mirror` only |
| Threshold tuning (pixels) | Edit `defaults.json5`; re-calibrate pixel fuzz |
| Schema break | Bump `schemaVersion`; migrate manifests |
